使用说明

打开方式

直接用浏览器打开本目录中的 index.html。所有 CSS、JS 和渲染逻辑都在本地，不需要 Kroki、CDN、远程字体或网络连接。

实时编辑

双击右侧预览图中的字段文字，会在图片上打开编辑框。输入内容会实时同步到左侧 PacketDiag 源码，并立即重绘预览。按 Enter 或点击其他位置结束编辑，按 Esc 取消本次修改。

基础语法

packetdiag {
  colwidth = 32;
  node_height = 40;
  default_fontsize = 12;
  bit_order = "desc";
  numbering = "global";

  // ---- Section Title ----
  // @left: 这段文字会显示在下一条字段所在行左侧
  0-15: Source Port [color = "#dbeafe"];
  16-31: Destination Port [textcolor = "red"];
}

连续编号 vs 每行独立编号

连续编号 (global, 默认)：位号全局唯一，字段按绝对位号跨行布局。

packetdiag {
  colwidth = 32;
  numbering = "global";   // 默认值，可省略

  // Row 0
  0-15: Source Port;
  16-31: Dest Port;
  // 下一个字段从 32 开始，自动进入下一行
  32-63: Sequence Number;
}

每行独立编号 (local)：每行独立从 0 开始编号，配合 // @row 显式分行。

packetdiag {
  colwidth = 32;
  numbering = "local";

  0-15: Source Port;
  16-31: Dest Port;

  // @row: Word 1
  0-31: Sequence Number;

  // @row: Word 2
  0-31: Ack Number;
}

工具栏「编号」下拉框可覆盖源码中的 numbering 配置；选择 每行独立 即可启用 local 模式。

⚠ 在 local 模式下，字段位号必须 < colwidth，超出会报错。

@row 行分隔

// @row 显式开始新视觉行：

// @row             — 开始新行，无标签
// @row: Word 1     — 开始新行，行标签 "Word 1"

第一行不需要写 // @row，从源码顶部开始。@row 在 global 模式下也会阻止自动行合并。

常用功能

  • 位序：工具栏可选择跟随源码、0 -> 31 或 31 -> 0；源码中可写 bit_order = "asc" 或 bit_order = "desc"。
  • 编号模式：工具栏可选择跟随源码、连续编号 或 每行独立；源码中可写 numbering = "global" 或 numbering = "local"。
  • 左侧注释：全局注释输入框显示在图片左侧顶部；源码中 // @left: 注释文字 会显示在下一条字段所在行左侧。
  • 字段样式：支持 color、textcolor、rotate = 270、colheight。
  • 导出：点击导出 PNG，会包含当前字段文字、位序、全局注释和行注释。

排查

  • 注释没有显示：确认 // @left: 写在目标字段前面。
  • 位序没有反向：确认工具栏位序为 跟随源码 且源码写了 bit_order = "desc"，或直接在工具栏选择 31 -> 0。
  • 编号模式不对：确认工具栏编号为 跟随源码 且源码写了 numbering = "local"，或直接在工具栏选择 每行独立。
  • 中文乱码：确认文件以 UTF-8 保存。
