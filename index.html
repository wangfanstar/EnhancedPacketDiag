PacketDiag 语法参考

快速入门

PacketDiag 用纯文本描述网络报文的位级结构，可生成报文头部示意图。

packetdiag {
  colwidth = 32;
  node_height = 40;

  // 这是注释
  0-31: 字段名称 [color = "#hex颜色"];
}

基本结构

所有内容必须包裹在 packetdiag { … } 块中。

配置参数

参数说明默认值示例
colwidth列宽比例32colwidth = 32;
node_height单元格高度(px)72node_height = 40;
default_fontsize默认字体大小12default_fontsize = 13;

位范围定义

每行定义一个字段，格式为 start-end: 字段名 [选项]：

0-3:   Version   [color = "#a8d8ea"];
4-7:   IHL;
8-15:  "带空格的标签";
16-31: Total Length;

单个比特位可以省略结束值：

154: NS_E;
203: R;

字段选项

选项说明示例
color单元格背景色[color = "#fce5cd"]
textcolor文字颜色[textcolor = "red"]

支持的颜色格式：十六进制（#rrggbb）、CSS 颜色名（red、lightblue）。

注释

使用 // 开头的行作为注释：

// 这是注释
0-31: FIELD;  // 行尾注释

区段标题

用 // ---- 分隔不同区段，区段之间会有视觉分隔：

// ---- IPv4 头部 ----
0-3: Version;
...

// ---- TCP 头部 ----
0-15: Source Port;
...

配置参数（完整）

参数说明可选值示例
colwidth列宽比例数字colwidth = 32;
node_height单元格高度(px)数字node_height = 40;
default_fontsize默认字体大小数字default_fontsize = 12;
numbering编号模式"global" / "local"numbering = "local";
bit_order位序方向"asc" / "desc"bit_order = "desc";

连续编号 vs 每行独立编号

连续编号 (global)：位号全局唯一，字段按绝对位号布局（默认模式）。

packetdiag {
  colwidth = 32;
  numbering = "global";   // 默认值，可省略

  0-15:  Port;
  16-31: Dest Port;
  // 下一个字段从 32 开始
  32-63: Seq Number;
}

每行独立编号 (local)：每行独立编号，多行可重复使用 0-31。配合 // @row 显式分行。

packetdiag {
  colwidth = 32;
  numbering = "local";

  // @row: Word 1
  0-15:  Port;
  16-31: Dest Port;

  // @row: Word 2
  0-31:  Seq Number;

  // @row: Word 3
  0-31:  Ack Number;
}

⚠ 在 local 模式下，字段位号必须 < colwidth，超出会报错。

位序 (bit_order)

  • bit_order = "asc"（默认）：位号从左到右递增（0 在左，31 在右）
  • bit_order = "desc"：位号从右到左递增（31 在左，0 在右），与许多协议规范图一致

@row 行分隔

// @row 显式开始新视觉行：

// @row             — 开始新行，无标签
// @row: Word 1     — 开始新行，行标签 "Word 1"

第一行不需要写 // @row，从源码顶部开始。

@left 行注释

// @left: 文字 在下一行左侧显示注释文字：

// @left: ← 这是源端口
0-15: Source Port;

变体 / 备选格式

用 变体、variant 或 alternative 关键字标记同一比特位的不同解释方式，将渲染为独立行：

// 变体 1: BFD上送
256-287: BFD_DISCR [color = "#fce5cd"];

// 变体 2: MOD上送
288-319: MOD_FIELD [color = "#fce5cd"];

完整示例

packetdiag {
  colwidth = 32;
  node_height = 48;
  default_fontsize = 12;

  // ---- IPv4 Header (20 bytes) ----

  0-3:   Version         [color = "#a8d8ea"];
  4-7:   IHL             [color = "#a8d8ea"];
  8-15:  Type of Service [color = "#b5e8c3"];
  16-31: Total Length    [color = "#f5d6a8"];

  32-47: Identification  [color = "#d5c4e8"];
  48-51: Flags           [color = "#f5f0b0"];
  52-63: Fragment Offset [color = "#f5f0b0"];

  64-71: TTL             [color = "#f5b0b0"];
  72-79: Protocol        [color = "#f5b0b0"];
  80-95: Header Checksum [color = "#d8d8d8"];

  96-127:  Source Address       [color = "#a0c8f0"];
  128-159: Destination Address  [color = "#a0c8f0"];
}

技巧

  • 位范围必须使用连续编号，从小到大排列
  • 带特殊字符的标签用双引号包裹，如 "Ethernet Type"
  • 区段之间的比特位范围可以不同（如 UDP 0–63, TCP 0–255）
  • 如果未指定颜色，系统会自动循环使用调色板
  • 工具栏「编号」和「位序」下拉框可覆盖源码配置，选择「跟随源码」即使用源码中的设置
  • 双击预览图中任意字段可编辑其标签文字，修改自动同步回源码
  • 导出支持 PNG（位图）、SVG（矢量图）和 HTML（离线可打开）三种格式
