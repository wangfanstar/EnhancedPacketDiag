PacketDiag Offline Viewer
PacketDiag 源码
渲染预览 双击字段可直接修改文字

使用说明

打开方式

开发版打开根目录 index.html；离线单文件版运行 npm run build 后打开 dist/index.html，可在 file:// 下直接使用。

标准 PacketDiag 语法

基于 blockdiag/packetdiag 语法。源码包裹在 packetdiag { } 内，也支持 @startpacketdiag / @endpacketdiag。

全局配置

参数默认值说明
colwidth32每行位宽，超出自动折行
node_height72字段单元格高度 (px)
default_fontsize12字段标签字号
scale_interval8标尺主刻度间隔

字段定义

格式：bit[-end]: 标签 [属性, …]

0-3:   Version [color = "#dbeafe"];
4-7:   IHL;
8:     Single Bit;
16-31: Total Len [textcolor = "red"];
96-111: "0x8100";                // 特殊字符标签用引号

字段属性

属性说明示例
color背景色[color = "#fce5cd"]
textcolor文字颜色[textcolor = "red"]
rotate文字旋转 90/270[rotate = 270]
colheight高度倍数 1–12[colheight = 3]
label显示别名[label = "Src Port"]
number是否显示位号徽章[number = 0]
description鼠标悬停提示[description = "RFC 793"]
style边框: solid/dashed/dotted/none[style = "dashed"]
shape形状: box/ellipse[shape = "ellipse"]
len变长字段，锯齿边缘[len = 64]
icon右上角图标徽章[icon = "flag"]
background背景色（优先 color）[background = "#c7d2fe"]

注释与区段

# 井号注释（标准 PacketDiag）
// 双斜杠注释
0-31: Field;  // 行尾注释

// ---- 区段标题 ----
// 变体 1: BFD上送        — 备选格式，渲染为独立行
// @row: Word 1           — 显式行分隔 [本工具独有]
// @left: 行注释          — 行左侧注释 [本工具独有]

desctable 描述表

packetdiag {
  0-15: SrcPort [description = "源端口"];
  desctable {
    SrcPort = "发送方端口号（16 bits）"
  }
}

本工具扩展功能

功能说明
numbering = "local"每行从 0 开始编号，可重复使用 0-31
bit_order = "desc"位序反转（31 在左，0 在右），匹配 IEEE 规范图
// @row / // @row: 标签显式行分隔，可带行标签
// @left: 文字下一行左侧显示注释
双击编辑双击预览图字段直接编辑标签，自动同步回源码
PNG 导出完全离线导出 PNG，无需 Kroki
全局注释框编辑器上方的文本框，显示在导出图左侧，不写回源码

工具栏「位序」「编号」下拉框可覆盖源码配置。

排查

  • 注释未显示：确认 // @left: 写在目标字段前面
  • 位序/编号不对：检查工具栏下拉框或源码 bit_order / numbering 配置
  • local 模式报错：所有字段位号需 < colwidth
  • 中文乱码：确认文件 UTF-8 编码
  • 页面空白：确认 assets/ 目录完整，或使用 dist/index.html
就绪 完全离线: 不请求 Kroki、CDN 或远程字体
