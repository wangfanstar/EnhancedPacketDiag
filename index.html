使用说明

打开方式

直接用浏览器打开本目录中的 index.html。所有 CSS、JS 和渲染逻辑都在本地，不需要 Kroki、CDN、远程字体或网络连接。

标准 PacketDiag 语法参考

本工具基于 blockdiag/packetdiag 语法，以下为完整的标准语法说明。

基本结构

PacketDiag 源码包裹在 packetdiag { … } 块内。标准语法还支持 @startpacketdiag / @endpacketdiag 包裹（PlantUML 风格），但本工具暂不支持该格式。

packetdiag {
  全局配置 ...
  位范围定义 ...
}

全局配置参数

参数类型默认值说明
colwidth整数32每行显示的位宽。字段位号超出 colwidth 时自动折行到下一行
node_height整数 (px)72每个字段单元格的高度
default_fontsize整数12字段标签的字体大小

位范围定义（标准语法）

每行定义一个字段，格式：bit[-end]: 标签 [属性, …]

0-3:   Version;                        // 多位字段
4-7:   IHL         [color = "#dbeafe"];  // 带颜色
8:     Single Bit;                        // 单个比特
16-31: Total Len   [textcolor = "red"];   // 文字颜色
96-111: "0x8100";                         // 含数字/特殊字符的标签需用引号包裹

标准字段属性

属性类型说明示例
color颜色值单元格背景色。支持 #rrggbb 或 CSS 颜色名[color = "#fce5cd"]
textcolor颜色值标签文字颜色[textcolor = "red"]
rotate90 / 270文字旋转角度。窄高字段自动旋转[rotate = 270]
colheight整数单元格高度倍数（1-12）[colheight = 3]

注释

标准 PacketDiag 使用 // 行注释（也兼容 #，本工具不支持 #）：

// 独立注释行
0-31: Field;  // 行尾注释

区段标题

使用 // ---- 标题 ---- 创建带背景色的区段分隔标题：

// ---- IPv4 Header ----
0-3: Version;
4-7: IHL;

// ---- TCP Header ----
0-15: Source Port;
...

变体 / 备选格式（标准语法）

标准 PacketDiag 中用 // 变体、// variant 或 // alternative 标记同一比特位的不同解释方式，渲染为独立行：

// 变体 1: BFD上送
256-287: BFD_DISCR [color = "#fce5cd"];

// 变体 2: MOD上送
288-311: MOD_FIELD [color = "#fce5cd"];

本工具扩展功能（与标准 PacketDiag 的差异）

以下功能为本工具独有，标准 PacketDiag / blockdiag / Kroki 中不存在：

1. numbering — 编号模式

取值行为
"global"（默认）标准模式：位号全局唯一，32-63 自动排在第二行
"local"每行独立模式：每行从 0 开始编号，多行可重复使用 0-31
packetdiag {
  colwidth = 32;
  numbering = "local";   // 本工具独有

  0-15: Source Port;
  16-31: Dest Port;

  // @row: Word 1      // 本工具独有
  0-31: Sequence Number;
}

工具栏「编号」下拉框可覆盖源码配置。

2. @row — 显式行分隔

标准 PacketDiag 通过空白行或字段位号的 colwidth 边界自动分行。本工具新增 // @row 显式行分隔：

// @row                — 开始新视觉行，不显示标签
// @row: Word 1        — 开始新视觉行，标签显示为 "Word 1"

第一行不需要写 // @row。在 global 模式下 @row 也会阻止自动行合并。

3. @left — 行注释

// @left: 这段注释会显示在下一行左侧
0-15: Source Port;

全局注释输入框的内容显示在图片左侧顶部，不写回源码。

4. bit_order — 位序反转

取值行为
"asc"（默认）0 在左，31 在右
"desc"31 在左，0 在右（匹配 IEEE/网络协议规范图习惯）

工具栏「位序」下拉框可覆盖源码配置。

5. 双击编辑

双击预览图中任意字段，可在图片上直接编辑标签文字，修改自动同步回源码。只改变 label，不会修改 bit 范围、颜色等属性。

6. 导出

标准 PacketDiag 依赖 Kroki 在线服务导出。本工具完全离线，支持导出 PNG 图片。

7. 全局注释输入框

源码编辑器上方的文本区域，内容显示在导出图片左侧顶部，不写回源码。

排查

  • 注释没有显示：确认 // @left: 写在目标字段前面。
  • 位序没有反向：确认工具栏「位序」为 跟随源码 且源码中写了 bit_order = "desc"，或直接在工具栏选 31 -> 0。
  • 编号模式不对：确认工具栏「编号」为 跟随源码 且源码写了 numbering = "local"，或直接在工具栏选 每行独立。
  • local 模式报错：确认所有字段位号 < colwidth。
  • 中文乱码：确认文件以 UTF-8 保存。
  • 页面空白：确认 index.html 和 assets/ 目录在同一位置。
就绪 完全离线: 不请求 Kroki、CDN 或远程字体
